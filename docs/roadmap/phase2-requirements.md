# yNet 第二轮迭代需求分析

## 一、当前系统的问题（Phase 1 回顾）

### 1.1 架构缺陷
```
当前架构：
┌─────────────┐      SSH隧道      ┌─────────────┐
│   本地节点   │ ───────────────→ │  远程Ollama  │
│  (你的Mac)   │                  │ (RTX 5060)   │
└─────────────┘                  └─────────────┘

问题：
❌ 单点依赖 - 只能调用一个远程节点
❌ 无负载均衡 - 无法选择最优节点
❌ 无容错机制 - 节点挂了就不可用
❌ 非真正分布式 - 只是远程调用，不是分布式计算
```

### 1.2 缺失的核心能力
- **模型分片**：无法把大模型切分到多个 GPU
- **流水线并行**：无法让多个节点协同处理一个请求
- **动态路由**：无法根据负载自动选择节点
- **容错恢复**：节点失败时无法自动切换

## 四、技术方案设计

### 4.1 分布式推理流程

```
用户请求流程：

1. 用户发起推理请求
   ↓
2. yNet 网关接收请求
   ↓
3. 查询节点注册表，获取可用节点列表
   {
     nodes: [
       { id: "A", gpu: "RTX4090", model: "qwen-72b", price: 0.0001, load: 0.3, latency: 50ms },
       { id: "B", gpu: "A100", model: "qwen-72b", price: 0.0002, load: 0.1, latency: 80ms },
       { id: "C", gpu: "M1Max", model: "qwen-7b", price: 0.00005, load: 0.5, latency: 120ms },
     ]
   }
   ↓
4. 智能路由选择最优节点
   - 规则 1: 模型必须匹配
   - 规则 2: 负载 < 80%
   - 规则 3: 按价格+延迟综合排序
   ↓
5. 发送请求到选中节点
   ↓
6. 节点执行推理，返回结果
   ↓
7. 自动结算费用
   - 用户账户扣除 X YNET
   - 节点账户增加 X * 0.95 YNET (5% 手续费)
   ↓
8. 返回结果给用户
```

### 4.3 智能路由算法

```rust
// 伪代码
fn select_best_node(nodes: Vec<Node>, request: InferenceRequest) -> Option<Node> {
    nodes
        .into_iter()
        // 1. 过滤：模型必须支持
        .filter(|n| n.has_model(&request.model))
        // 2. 过滤：负载不能太高
        .filter(|n| n.load < 0.8)
        // 3. 过滤：最近有响应
        .filter(|n| n.last_seen < Duration::from_secs(30))
        // 4. 排序：综合评分
        .min_by(|a, b| {
            let score_a = calculate_score(a, &request.preferences);
            let score_b = calculate_score(b, &request.preferences);
            score_a.cmp(&score_b)
        })
}

fn calculate_score(node: &Node, prefs: &Preferences) -> f64 {
    let price_weight = prefs.price_priority;      // 0.0 - 1.0
    let latency_weight = 1.0 - price_weight;

    // 归一化各指标
    let price_score = normalize(node.price, min_price, max_price);  // 0-1, 越小越好
    let latency_score = normalize(node.latency, min_latency, max_latency);  // 0-1, 越小越好
    let load_penalty = node.load;  // 越小越好

    price_weight * price_score + latency_weight * latency_score + 0.2 * load_penalty
}
```
